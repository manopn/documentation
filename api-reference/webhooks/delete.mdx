# Desativar um Webhook Registrado — DELETE /api/webhooks

**Descrição:** `DELETE /api/webhooks` — Desativa o endpoint de webhook da sua conta CertoPay. Você deixará de receber notificações de eventos de transação até que uma nova URL seja registrada.

---

Use este endpoint para desativar o webhook atualmente registrado. Uma vez excluído, a CertoPay interrompe imediatamente a entrega de notificações de eventos de transação para o seu endpoint. Isso é útil quando você precisa trocar a URL do webhook, pausar temporariamente a entrega de eventos durante uma manutenção, ou descomissionar um receptor de webhook que não está mais em uso.

---

## Endpoint

```
DELETE https://v2.certopaybrasil.com/api/webhooks
```

---

## Autenticação

Todas as requisições devem incluir sua chave de API secreta no cabeçalho `X-Api-Key`.

| Cabeçalho | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `X-Api-Key` | string | ✅ | Sua chave de API secreta da CertoPay (ex.: `sk_live_sua_chave_aqui`). |

---

## Exemplo de Requisição

```bash
curl -X DELETE https://v2.certopaybrasil.com/api/webhooks \
  -H "X-Api-Key: sk_live_sua_chave_aqui"
```

---

## Resposta

### 200 — Sucesso

Uma chamada bem-sucedida confirma que o webhook foi desativado.

```json
{
  "success": true,
  "message": "Webhook successfully deleted."
}
```

---

> ⚠️ **Atenção:** Após excluir o webhook, você **deixará de receber todas as notificações de eventos de transação** — incluindo eventos críticos como `transaction.paid` e `transaction.failed`. Qualquer evento que ocorra enquanto o webhook estiver inativo não será reenviado automaticamente. Para retomar o recebimento de notificações, registre uma nova URL usando `POST /api/webhooks/upsert`.
