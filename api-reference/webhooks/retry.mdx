# Reenviar uma Entrega de Webhook — POST /api/webhooks/deliveries

**Descrição:** `POST /api/webhooks/deliveries/{deliveryId}/retry` — Dispara manualmente o reenvio de uma entrega de webhook com falha. Útil para recuperar eventos de transação perdidos.

---

Use este endpoint para reenviar manualmente uma entrega de webhook que anteriormente falhou ou não foi confirmada pelo seu servidor. Ao iniciar um reenvio, a CertoPay envia novamente o payload do evento original para a URL de webhook atualmente registrada. Isso é especialmente útil para recuperar eventos críticos perdidos — como uma notificação `transaction.paid` que seu servidor não conseguiu processar devido a uma indisponibilidade temporária ou configuração incorreta.

---

## Endpoint

```
POST https://v2.certopaybrasil.com/api/webhooks/deliveries/{deliveryId}/retry
```

---

## Autenticação

Todas as requisições devem incluir sua chave de API secreta no cabeçalho `X-Api-Key`.

| Cabeçalho | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `X-Api-Key` | string | ✅ | Sua chave de API secreta da CertoPay (ex.: `sk_live_sua_chave_aqui`). |

---

## Parâmetros de Caminho

| Parâmetro | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `deliveryId` | string | ✅ | UUID único da entrega de webhook que você deseja reenviar. Recupere este valor no campo `deliveryId` da resposta de `GET /api/webhooks/deliveries`. |

---

## Exemplo de Requisição

```bash
curl -X POST https://v2.certopaybrasil.com/api/webhooks/deliveries/delivery-uuid/retry \
  -H "X-Api-Key: sk_live_sua_chave_aqui"
```

---

## Resposta

### 200 — Sucesso

Uma chamada bem-sucedida confirma que o reenvio foi enfileirado e o payload do evento original será reenviado para a URL de webhook registrada.

```json
{
  "success": true,
  "deliveryId": "delivery-uuid",
  "message": "Retry queued successfully."
}
```

### Campos da Resposta

| Campo | Tipo | Descrição |
|---|---|---|
| `success` | boolean | `true` quando o reenvio foi aceito e enfileirado para entrega. |
| `deliveryId` | string | UUID do registro de entrega que foi enfileirado para reenvio. |
| `message` | string | Mensagem de confirmação legível. |

---

> 📝 **Nota:** Para encontrar o `deliveryId` de uma entrega com falha, chame `GET /api/webhooks/deliveries` filtrando por `status=FAILED`. Copie o `deliveryId` de qualquer entrega que deseje reenviar e passe-o como parâmetro de caminho para este endpoint.
