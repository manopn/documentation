# GET /api/webhooks/deliveries — Histórico de Entregas de Webhooks

**Descrição:** `GET /api/webhooks/deliveries` — Visualize o histórico de entregas dos seus webhooks CertoPay. Filtre por status ou tipo de evento para depurar entregas com falha.

---

Este endpoint retorna um log de todas as tentativas de entrega de webhook feitas pela CertoPay ao seu endpoint registrado. Cada registro captura o tipo de evento, o resultado da entrega, a resposta HTTP recebida do seu servidor e o horário em que a entrega foi tentada. Use os parâmetros de consulta opcionais para filtrar por status de entrega ou tipo de evento — por exemplo, para exibir apenas as entregas com falha que precisam de atenção ou reenvio.

---

## Endpoint

```
GET https://v2.certopaybrasil.com/api/webhooks/deliveries
```

---

## Autenticação

Todas as requisições devem incluir sua chave de API secreta no cabeçalho `X-Api-Key`.

| Cabeçalho | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `X-Api-Key` | string | ✅ | Sua chave de API secreta da CertoPay (ex.: `sk_live_sua_chave_aqui`). |

---

## Parâmetros de Consulta

| Parâmetro | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `status` | string | ❌ | Filtra as entregas pelo resultado. Valores aceitos: `SUCCESS`, `FAILED`, `PENDING`. Omita este parâmetro para retornar entregas de todos os status. |
| `event` | string | ❌ | Filtra as entregas pelo tipo de evento que as disparou (ex.: `transaction.paid`, `transaction.failed`, `transaction.refunded`). Omita este parâmetro para retornar entregas de todos os tipos de evento. |

---

## Exemplos de Requisição

Recuperar todas as entregas com falha:
```bash
curl "https://v2.certopaybrasil.com/api/webhooks/deliveries?status=FAILED" \
  -H "X-Api-Key: sk_live_sua_chave_aqui"
```

Recuperar todas as entregas de eventos `transaction.paid`:
```bash
curl "https://v2.certopaybrasil.com/api/webhooks/deliveries?event=transaction.paid" \
  -H "X-Api-Key: sk_live_sua_chave_aqui"
```

Combinar filtros para recuperar entregas de `transaction.paid` com falha:
```bash
curl "https://v2.certopaybrasil.com/api/webhooks/deliveries?status=FAILED&event=transaction.paid" \
  -H "X-Api-Key: sk_live_sua_chave_aqui"
```

---

## Resposta

Retorna um array de objetos de entrega de webhook ordenados pela tentativa mais recente primeiro.

### 200 — Sucesso

```json
[
  {
    "deliveryId": "uuid-da-entrega",
    "event": "transaction.paid",
    "status": "FAILED",
    "httpStatus": 500,
    "attemptedAt": "2026-06-26T10:05:30Z"
  },
  {
    "deliveryId": "uuid-da-entrega-2",
    "event": "transaction.created",
    "status": "SUCCESS",
    "httpStatus": 200,
    "attemptedAt": "2026-06-26T10:00:05Z"
  }
]
```

### Campos da Resposta

| Campo | Tipo | Descrição |
|---|---|---|
| `deliveryId` | string | Identificador único (UUID) desta tentativa de entrega. Use este valor com `POST /api/webhooks/deliveries/{id}/retry` para reenviar manualmente uma entrega com falha. |
| `event` | string | Tipo do evento que disparou esta entrega (ex.: `transaction.paid`, `transaction.failed`). |
| `status` | string | Resultado da tentativa de entrega. Valores possíveis: `SUCCESS`, `FAILED`, `PENDING`. |
| `httpStatus` | integer | Código de status HTTP retornado pelo seu endpoint quando a CertoPay tentou a entrega (ex.: `200`, `500`, `404`). Este campo pode ser `null` se a requisição nunca chegou ao seu servidor (ex.: devido a um erro de DNS ou de conexão). |
| `attemptedAt` | string | Timestamp ISO 8601 de quando esta entrega foi tentada. |

---

> 📝 **Nota:** Use este endpoint em conjunto com `POST /api/webhooks/deliveries/{id}/retry` para recuperar eventos perdidos. Recupere o `deliveryId` de qualquer entrega com falha nesta lista e passe-o ao endpoint de reenvio para disparar novamente a notificação.
