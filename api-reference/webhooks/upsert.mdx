# Registrar ou Atualizar um Webhook — POST /api/webhooks/upsert

**Descrição:** `POST /api/webhooks/upsert` — Registra ou atualiza sua URL de webhook para receber notificações de eventos de transação em tempo real da CertoPay.

---

Use este endpoint para registrar uma URL HTTPS que a CertoPay chamará sempre que um evento de transação ocorrer na sua conta — como a confirmação de um pagamento, o vencimento de um boleto ou o processamento de um reembolso. A CertoPay envia uma requisição `POST` com um payload JSON para a sua URL cada vez que um evento qualificado é disparado. Se você já tiver uma URL de webhook registrada, chamar este endpoint novamente irá atualizá-la com o novo valor (comportamento de upsert).

---

## Endpoint

```
POST https://v2.certopaybrasil.com/api/webhooks/upsert
```

---

## Autenticação

Todas as requisições devem incluir sua chave de API secreta no cabeçalho `X-Api-Key`.

| Cabeçalho | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `X-Api-Key` | string | ✅ | Sua chave de API secreta da CertoPay (ex.: `sk_live_sua_chave_aqui`). |
| `Content-Type` | string | ✅ | Deve ser `application/json`. |

---

## Corpo da Requisição

| Campo | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `url` | string | ✅ | URL HTTPS completa do seu endpoint de webhook. A CertoPay entregará as notificações de eventos de transação para este endereço. Deve utilizar o esquema `https://` — URLs em HTTP simples não são aceitas. |

---

## Exemplo de Requisição

```bash
curl -X POST https://v2.certopaybrasil.com/api/webhooks/upsert \
  -H "X-Api-Key: sk_live_sua_chave_aqui" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://seusite.com/webhooks/certopay"}'
```

## Exemplo de Corpo da Requisição

```json
{
  "url": "https://seusite.com/webhooks/certopay"
}
```

---

## Resposta

### 200 — Sucesso

Uma chamada bem-sucedida retorna uma confirmação de que a URL do webhook foi registrada ou atualizada.

```json
{
  "success": true,
  "url": "https://seusite.com/webhooks/certopay"
}
```

---

> 📝 **Nota:** Este endpoint utiliza comportamento de upsert — chamá-lo novamente com uma URL diferente atualizará o registro de webhook existente em vez de criar uma duplicata. Há apenas uma URL de webhook ativa por conta. Para visualizar a URL atualmente registrada, use `GET /api/webhooks`.
