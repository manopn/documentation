# GET /api/webhooks — Recuperar Configuração do Webhook

**Descrição:** `GET /api/webhooks` — Recupera a URL do webhook registrada na sua conta CertoPay e seu status atual. Confirme que a configuração está correta antes de testar qualquer fluxo de pagamento.

---

Use este endpoint para inspecionar sua configuração atual de webhook. A resposta inclui a URL registrada para receber notificações de eventos de transação e seu status ativo. Isso é útil para verificar se o webhook está configurado corretamente antes de testar um fluxo de pagamento, ou para confirmar qual endpoint está registrado após realizar alterações via `POST /api/webhooks/upsert`.

---

## Endpoint

```
GET https://v2.certopaybrasil.com/api/webhooks
```

---

## Autenticação

Todas as requisições devem incluir sua chave de API secreta no cabeçalho `X-Api-Key`.

| Cabeçalho | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `X-Api-Key` | string | ✅ | Sua chave de API secreta da CertoPay (ex.: `sk_live_sua_chave_aqui`). |

---

## Exemplo de Requisição

```bash
curl https://v2.certopaybrasil.com/api/webhooks \
  -H "X-Api-Key: sk_live_sua_chave_aqui"
```

---

## Resposta

Retorna o objeto de configuração do webhook da sua conta, incluindo a URL registrada e seu status atual.

### 200 — Sucesso

```json
{
  "id": "uuid-do-webhook",
  "url": "https://seusite.com/webhooks/certopay",
  "status": "ACTIVE",
  "createdAt": "2026-06-20T08:00:00Z",
  "updatedAt": "2026-06-26T09:00:00Z"
}
```

### Campos da Resposta

| Campo | Tipo | Descrição |
|---|---|---|
| `id` | string | Identificador único (UUID) da sua configuração de webhook. |
| `url` | string | A URL HTTPS atualmente registrada para receber notificações de eventos de transação. |
| `status` | string | Status atual do webhook. `ACTIVE` significa que as notificações estão sendo entregues; `INACTIVE` significa que o webhook foi desativado. |
| `createdAt` | string | Timestamp ISO 8601 de quando o webhook foi registrado pela primeira vez. |
| `updatedAt` | string | Timestamp ISO 8601 de quando a configuração do webhook foi modificada pela última vez. |

---

> 📝 **Nota:** Se nenhum webhook foi registrado na sua conta, este endpoint pode retornar um `404` ou uma resposta vazia. Use `POST /api/webhooks/upsert` para registrar uma URL e começar a receber notificações.
