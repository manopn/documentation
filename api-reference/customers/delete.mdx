# Excluir Registro de Cliente — DELETE /api/customers/{id}

**DELETE /api/customers/{id} — Remove permanentemente um cliente da sua conta CertoPay. Esta ação é irreversível e não pode ser desfeita.**

Use este endpoint para excluir permanentemente um registro de cliente da sua conta CertoPay. Uma vez excluído, o UUID do cliente não é mais válido e não pode ser usado em pedidos ou solicitações de pagamento. Esta ação é tipicamente utilizada para requisições de exclusão de dados da LGPD ou para limpar dados de teste da sua conta. Passe o UUID do cliente na URL onde `{id}` aparece.

> ⚠️ **Atenção: Esta ação é irreversível.** Excluir um cliente remove permanentemente seu registro do CertoPay. Quaisquer pedidos ou pagamentos anteriormente associados a este cliente manterão seus dados históricos, mas não será mais possível criar novas transações para este ID de cliente. Não há como recuperar um cliente excluído — seria necessário cadastrá-lo novamente via `POST /api/customers`.

---

## Requisição

| Parâmetro | Local | Tipo | Obrigatório | Descrição |
|---|---|---|---|---|
| `X-Api-Key` | cabeçalho | string | ✅ | Sua chave secreta de API CertoPay. Inclua em toda solicitação como `X-Api-Key: sk_live_sua_chave_aqui`. |
| `id` | caminho (path) | string | ✅ | O UUID do cliente a ser excluído permanentemente. É o `id` retornado quando o cliente foi criado. |

Este endpoint não aceita corpo na requisição.

---

## Resposta

Uma solicitação bem-sucedida retorna **HTTP 204 No Content** com corpo de resposta vazio, confirmando que o cliente foi excluído.

---

## Exemplo

```bash
curl -X DELETE https://v2.certopaybrasil.com/api/customers/uuid-do-customer \
  -H "X-Api-Key: sk_live_sua_chave_aqui"
```

```text
204 No Content
(corpo vazio)
```
