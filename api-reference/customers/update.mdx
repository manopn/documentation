# Atualizar Campos do Cliente — PATCH /api/customers/{id}

**PATCH /api/customers/{id} — Atualiza parcialmente os dados de um cliente no CertoPay. Envie apenas os campos que deseja alterar; todos os outros permanecem iguais.**

Use este endpoint para atualizar parcialmente um registro de cliente existente. Por ser uma requisição `PATCH`, você só precisa incluir os campos que deseja modificar — todos os campos omitidos permanecerão inalterados. É ideal para corrigir um número de telefone, atualizar um endereço de e-mail ou ajustar um nome sem precisar reenviar todo o payload do cliente. Passe o UUID do cliente na URL onde `{id}` aparece.

---

## Requisição

| Parâmetro | Local | Tipo | Obrigatório | Descrição |
|---|---|---|---|---|
| `X-Api-Key` | cabeçalho | string | ✅ | Sua chave secreta de API CertoPay. Inclua em toda solicitação como `X-Api-Key: sk_live_sua_chave_aqui`. |
| `id` | caminho (path) | string | ✅ | O UUID do cliente a ser atualizado. É o `id` retornado quando o cliente foi criado. |
| `name` | corpo | string | — | Nome completo atualizado do cliente. Omita este campo para manter o nome inalterado. |
| `email` | corpo | string | — | Endereço de e-mail atualizado. Omita este campo para manter o e-mail inalterado. |
| `phone` | corpo | string | — | Número de telefone atualizado no formato brasileiro (ex.: `"(11) 88888-8888"`). Omita este campo para manter o telefone inalterado. |

---

## Resposta

Uma solicitação bem-sucedida retorna **HTTP 200 OK** com o objeto completo e atualizado do cliente, refletindo todos os valores atuais dos campos.

| Campo | Tipo | Descrição |
|---|---|---|
| `id` | string | UUID único que identifica o cliente (inalterado). |
| `name` | string | Nome completo — atualizado se fornecido na requisição, caso contrário mantém o valor anterior. |
| `email` | string | Endereço de e-mail — atualizado se fornecido na requisição, caso contrário mantém o valor anterior. |
| `document` | string | CPF (11 dígitos). Este campo não pode ser alterado via PATCH. |
| `phone` | string | Número de telefone — atualizado se fornecido na requisição, caso contrário mantém o valor anterior. |
| `createdAt` | string | Timestamp ISO 8601 da data de criação original (inalterado). |

---

## Exemplo

```bash
curl -X PATCH https://v2.certopaybrasil.com/api/customers/uuid-do-customer \
  -H "X-Api-Key: sk_live_sua_chave_aqui" \
  -H "Content-Type: application/json" \
  -d '{"phone": "(11) 88888-8888"}'
```

```json
{
  "id": "uuid-do-customer",
  "name": "João da Silva",
  "email": "joao@email.com",
  "document": "12345678900",
  "phone": "(11) 88888-8888",
  "createdAt": "2026-06-26T10:00:00Z"
}
```
