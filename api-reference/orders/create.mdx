# Criar Novo Pedido de Pagamento — POST /api/orders

**POST /api/orders — Cria um pedido de pagamento no CertoPay vinculando um cliente a um valor em centavos. Retorna o orderId para processamento do pagamento.**

Use este endpoint para criar um novo pedido de pagamento no CertoPay. Um pedido representa a intenção comercial de cobrar um valor específico de um cliente cadastrado, e deve ser criado antes que qualquer método de pagamento (Pix, boleto ou cartão de crédito) possa ser aplicado. Você deve fornecer um `customerId` válido obtido via `POST /api/customers`, junto com o valor da cobrança expresso em **centavos** (a menor unidade monetária brasileira). O `id` do pedido retornado se torna o `orderId` que você referenciará ao processar o pagamento de fato.

---

## Requisição

| Parâmetro | Local | Tipo | Obrigatório | Descrição |
|---|---|---|---|---|
| `X-Api-Key` | cabeçalho | string | ✅ | Sua chave secreta de API CertoPay. Inclua em toda solicitação como `X-Api-Key: sk_live_sua_chave_aqui`. |
| `customerId` | corpo | string | ✅ | O UUID do cliente a ser cobrado. É o `id` retornado por `POST /api/customers`. O cliente já deve existir na sua conta. |
| `amount` | corpo | integer | ✅ | Valor total da cobrança em **centavos**. Por exemplo, `29700` representa R$ 297,00. Deve ser um número inteiro positivo maior que zero. |

---

## Resposta

Uma solicitação bem-sucedida retorna **HTTP 201 Created** com o objeto do pedido recém-criado.

| Campo | Tipo | Descrição |
|---|---|---|
| `id` | string | UUID único atribuído a este pedido. **Guarde este valor** — ele é obrigatório como `orderId` em todas as requisições de pagamento posteriores. |
| `customerId` | string | UUID do cliente vinculado a este pedido, repetido da requisição. |
| `amount` | integer | Valor da cobrança em centavos, repetido da requisição. |
| `status` | string | Status atual do pedido. Pedidos recém-criados sempre iniciam como `"PENDING"`. Outros valores possíveis conforme o pedido avança: `"PAID"`, `"CANCELLED"`, `"EXPIRED"`. |
| `createdAt` | string | Timestamp ISO 8601 registrando quando o pedido foi criado (ex.: `"2026-06-26T10:00:00Z"`). |

> ℹ️ **Nota:** Guarde o `id` retornado nesta resposta — ele se torna o `orderId` obrigatório em todas as requisições de pagamento subsequentes (Pix, boleto e cartão de crédito). Sem ele não é possível cobrar o cliente.

---

## Exemplo

```bash
curl -X POST https://v2.certopaybrasil.com/api/orders \
  -H "X-Api-Key: sk_live_sua_chave_aqui" \
  -H "Content-Type: application/json" \
  -d '{
    "customerId": "uuid-do-customer",
    "amount": 29700
  }'
```

```json
{
  "id": "uuid-do-pedido",
  "customerId": "uuid-do-customer",
  "amount": 29700,
  "status": "PENDING",
  "createdAt": "2026-06-26T10:00:00Z"
}
```
