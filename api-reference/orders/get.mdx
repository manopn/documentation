# Consultar Pedido por UUID — GET /api/orders/{id}

**GET /api/orders/{id} — Busca um único pedido pelo UUID. Retorna os detalhes do pedido incluindo ID do cliente, valor, status e data de criação.**

Use este endpoint para buscar os detalhes completos de um único pedido pelo seu UUID. Esta é a principal forma de verificar se uma cobrança específica foi paga, ainda está pendente ou foi cancelada ou expirada. Também é útil quando você precisa confirmar o valor de um pedido ou o cliente vinculado antes de iniciar um fluxo de pagamento. Passe o UUID do pedido na URL onde `{id}` aparece.

---

## Requisição

| Parâmetro | Local | Tipo | Obrigatório | Descrição |
|---|---|---|---|---|
| `X-Api-Key` | cabeçalho | string | ✅ | Sua chave secreta de API CertoPay. Inclua em toda solicitação como `X-Api-Key: sk_live_sua_chave_aqui`. |
| `id` | caminho (path) | string | ✅ | O UUID do pedido a ser consultado. É o `id` retornado quando o pedido foi criado via `POST /api/orders`. |

Este endpoint não aceita corpo na requisição.

---

## Resposta

Uma solicitação bem-sucedida retorna **HTTP 200 OK** com o objeto completo do pedido.

| Campo | Tipo | Descrição |
|---|---|---|
| `id` | string | UUID único que identifica o pedido. |
| `customerId` | string | UUID do cliente vinculado a este pedido. |
| `amount` | integer | Valor da cobrança em centavos (ex.: `29700` = R$ 297,00). |
| `status` | string | Status atual do pagamento do pedido. Valores possíveis: `"PENDING"`, `"PAID"`, `"CANCELLED"`, `"EXPIRED"`. |
| `createdAt` | string | Timestamp ISO 8601 de quando o pedido foi criado. |

---

## Exemplo

```bash
curl https://v2.certopaybrasil.com/api/orders/uuid-do-pedido \
  -H "X-Api-Key: sk_live_sua_chave_aqui"
```

```json
{
  "id": "uuid-do-pedido",
  "customerId": "uuid-do-customer",
  "amount": 29700,
  "status": "PENDING",
  "createdAt": "2026-06-26T10:00:00Z"
}
```
