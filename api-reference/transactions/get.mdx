# GET /api/transactions/{transactionId} — Obter Transação

**Descrição:** `GET /api/transactions/{transactionId}` — Busca uma transação específica pelo ID. Confirme o status do pagamento antes de processar pedidos ou liberar acessos.

---

Use este endpoint para buscar os detalhes de uma transação específica pelo seu identificador único. Isso é especialmente importante em fluxos pós-pagamento — por exemplo, após receber uma notificação de webhook, você deve sempre recuperar a transação diretamente da API para confirmar seu status antes de processar um pedido ou liberar o acesso a um produto.

---

## Endpoint

```
GET https://v2.certopaybrasil.com/api/transactions/{transactionId}
```

---

## Autenticação

Todas as requisições devem incluir sua chave de API secreta no cabeçalho `X-Api-Key`.

| Cabeçalho | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `X-Api-Key` | string | ✅ | Sua chave de API secreta da CertoPay (ex.: `sk_live_sua_chave_aqui`). |

---

## Parâmetros de Caminho

| Parâmetro | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `transactionId` | string | ✅ | UUID único da transação que você deseja recuperar. Pode ser obtido pelo endpoint Listar Transações ou a partir de um payload de webhook. |

---

## Exemplo de Requisição

```bash
curl https://v2.certopaybrasil.com/api/transactions/uuid-da-transacao \
  -H "X-Api-Key: sk_live_sua_chave_aqui"
```

---

## Resposta

Retorna um único objeto de transação correspondente ao ID fornecido.

### 200 — Sucesso

```json
{
  "id": "uuid-da-transacao",
  "orderId": "uuid-do-pedido",
  "method": "PIX",
  "amount": 29700,
  "status": "PAID",
  "createdAt": "2026-06-26T10:00:00Z",
  "updatedAt": "2026-06-26T10:05:00Z"
}
```

### Campos da Resposta

| Campo | Tipo | Descrição |
|---|---|---|
| `id` | string | Identificador único (UUID) desta transação. |
| `orderId` | string | UUID do pedido associado a esta transação. |
| `method` | string | Método de pagamento utilizado. Valores possíveis: `PIX`, `BOLETO`, `CARD`. |
| `amount` | integer | Valor da transação em centavos (ex.: `29700` = R$ 297,00). |
| `status` | string | Status atual da transação (ex.: `PAID`, `PENDING`, `FAILED`, `REFUNDED`). |
| `createdAt` | string | Timestamp ISO 8601 de quando a transação foi criada. |
| `updatedAt` | string | Timestamp ISO 8601 de quando o registro da transação foi atualizado pela última vez. |

---

> 💡 **Dica:** Sempre chame este endpoint após receber um webhook `transaction.paid` para verificar o status antes de liberar o acesso. Nunca tome decisões de acesso baseando-se apenas no payload do webhook — confirme sempre o estado atual diretamente pela API.
