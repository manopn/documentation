# GET /api/transactions — Listar Todas as Transações de Pagamento

**Descrição:** `GET /api/transactions` — Recupera todas as transações de pagamento da sua conta CertoPay. Retorna um array com status, método, valor e timestamps.

---

Use este endpoint para recuperar a lista completa de transações de pagamento associadas à sua conta CertoPay. Cada objeto no array retornado contém informações de identificação, o método de pagamento utilizado, o valor em centavos, o status atual de processamento e os timestamps de criação e última atualização da transação. Esta é uma operação somente de leitura e não requer corpo na requisição.

---

## Endpoint

```
GET https://v2.certopaybrasil.com/api/transactions
```

---

## Autenticação

Todas as requisições devem incluir sua chave de API secreta no cabeçalho `X-Api-Key`. Mantenha esta chave em segurança e nunca a exponha em código do lado do cliente.

| Cabeçalho | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `X-Api-Key` | string | ✅ | Sua chave de API secreta da CertoPay (ex.: `sk_live_sua_chave_aqui`). |

---

## Exemplo de Requisição

```bash
curl https://v2.certopaybrasil.com/api/transactions \
  -H "X-Api-Key: sk_live_sua_chave_aqui"
```

---

## Resposta

Retorna um array de objetos de transação. Um array vazio (`[]`) é retornado quando ainda não existem transações.

### 200 — Sucesso

```json
[
  {
    "id": "uuid-da-transacao",
    "orderId": "uuid-do-pedido",
    "method": "PIX",
    "amount": 29700,
    "status": "PAID",
    "createdAt": "2026-06-26T10:00:00Z",
    "updatedAt": "2026-06-26T10:05:00Z"
  }
]
```

### Campos da Resposta

| Campo | Tipo | Descrição |
|---|---|---|
| `id` | string | Identificador único (UUID) desta transação. |
| `orderId` | string | UUID do pedido associado a esta transação. Use este campo para correlacionar uma transação com o pedido correspondente no seu sistema. |
| `method` | string | Método de pagamento utilizado na transação. Valores possíveis: `PIX`, `BOLETO`, `CARD`. |
| `amount` | integer | Valor da transação expresso em centavos (ex.: `29700` = R$ 297,00). Todos os valores monetários são inteiros para evitar problemas de arredondamento de ponto flutuante. |
| `status` | string | Status atual da transação (ex.: `PAID`, `PENDING`, `FAILED`, `REFUNDED`). |
| `createdAt` | string | Timestamp ISO 8601 indicando quando a transação foi criada (ex.: `2026-06-26T10:00:00Z`). |
| `updatedAt` | string | Timestamp ISO 8601 indicando quando o registro da transação foi atualizado pela última vez (ex.: `2026-06-26T10:05:00Z`). |
