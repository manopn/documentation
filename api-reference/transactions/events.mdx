# GET /api/transaction-events — Listar Eventos de Transação

**Descrição:** `GET /api/transaction-events/transaction/{id}` — Recupera o histórico completo de eventos de uma transação específica, exibindo cada mudança de status em ordem cronológica.

---

Este endpoint retorna o histórico completo de eventos de uma determinada transação, listando cada mudança de status em ordem cronológica. Cada registro de evento descreve uma transição de estado discreta — por exemplo, de `PENDING` para `PROCESSING` e depois para `PAID`. Essa trilha de auditoria é fundamental para depurar fluxos de pagamento, conciliar registros e investigar disputas ou resultados inesperados em transações.

---

## Endpoint

```
GET https://v2.certopaybrasil.com/api/transaction-events/transaction/{transactionId}
```

---

## Autenticação

Todas as requisições devem incluir sua chave de API secreta no cabeçalho `X-Api-Key`.

| Cabeçalho | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `X-Api-Key` | string | ✅ | Sua chave de API secreta da CertoPay (ex.: `sk_live_sua_chave_aqui`). |

---

## Parâmetros de Caminho

| Parâmetro | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `transactionId` | string | ✅ | UUID único da transação cujo histórico de eventos você deseja recuperar. Obtenha este valor pelos endpoints Listar Transações ou Obter Transação. |

---

## Exemplo de Requisição

```bash
curl https://v2.certopaybrasil.com/api/transaction-events/transaction/uuid-da-transacao \
  -H "X-Api-Key: sk_live_sua_chave_aqui"
```

---

## Resposta

Retorna um array de objetos de evento, ordenados cronologicamente do mais antigo para o mais recente. Cada objeto descreve uma única transição de status da transação.

### 200 — Sucesso

```json
[
  {
    "id": "uuid-do-evento",
    "transactionId": "uuid-da-transacao",
    "type": "transaction.created",
    "status": "PENDING",
    "createdAt": "2026-06-26T10:00:00Z"
  },
  {
    "id": "uuid-do-evento-2",
    "transactionId": "uuid-da-transacao",
    "type": "transaction.paid",
    "status": "PAID",
    "createdAt": "2026-06-26T10:05:00Z"
  }
]
```

### Campos da Resposta

| Campo | Tipo | Descrição |
|---|---|---|
| `id` | string | Identificador único (UUID) deste registro de evento individual. |
| `transactionId` | string | UUID da transação à qual este evento pertence. |
| `type` | string | Tipo do evento, descrevendo o que ocorreu (ex.: `transaction.created`, `transaction.paid`, `transaction.failed`). |
| `status` | string | Status da transação no momento em que este evento foi registrado (ex.: `PENDING`, `PROCESSING`, `PAID`, `FAILED`). |
| `createdAt` | string | Timestamp ISO 8601 de quando este evento foi registrado. |

---

> 📝 **Nota:** A lista de eventos é ordenada cronologicamente — o primeiro item representa o estado mais antigo registrado e o último representa o mais recente. Isso facilita o rastreamento do ciclo de vida completo de um pagamento, desde a criação até a conclusão ou falha. Use este endpoint em conjunto com o Histórico de Entrega de Webhooks para uma auditoria abrangente do fluxo de pagamentos.
