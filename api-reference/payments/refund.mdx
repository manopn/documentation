# POST /api/payment-gateway/refund — Emitir um Reembolso

**Descrição:** `POST /api/payment-gateway/refund` — Emite um reembolso total ou parcial para uma transação capturada. Reembolsos parciais exigem o campo `amount` em centavos.

---

O endpoint `/api/payment-gateway/refund` permite reverter uma transação capturada de forma total ou parcial. Reembolsos totais devolvem o valor integral cobrado ao comprador, enquanto reembolsos parciais permitem especificar um valor exato em centavos — útil em cenários como descontos retroativos, itens cancelados ou devoluções proporcionais. Reembolsos só podem ser emitidos para transações nos estados `CAPTURED` ou `PAID`.

---

## URL Base

```
https://v2.certopaybrasil.com/api
```

## Endpoint

```
POST /payment-gateway/refund
```

---

## Cabeçalhos da Requisição

| Cabeçalho | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `X-Api-Key` | string | ✅ | Sua chave de API secreta. Mantenha-a no servidor e nunca a exponha em código voltado ao cliente. |
| `Content-Type` | string | ✅ | Deve ser `application/json`. |

---

## Corpo da Requisição e Exemplos

### Reembolso Total

Um reembolso total reverte o valor integral da cobrança original. Omita o campo `amount` — a CertoPay usará automaticamente o valor total da transação.

**Campos da Requisição**

| Campo | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `transactionId` | string | ✅ | Identificador único da transação a ser reembolsada. É o `transactionId` retornado por `POST /api/payment-gateway/process`. |
| `reason` | string | ❌ | Explicação legível para o reembolso. Armazenada para fins de auditoria e visível no painel da CertoPay. Exemplo: `"Solicitação do cliente"`. |

**Exemplo de Requisição**

```bash
curl -X POST https://v2.certopaybrasil.com/api/payment-gateway/refund \
  -H "X-Api-Key: sk_live_sua_chave_aqui" \
  -H "Content-Type: application/json" \
  -d '{
    "transactionId": "uuid-da-transacao",
    "reason": "Solicitação do cliente"
  }'
```

**Campos da Resposta**

| Campo | Tipo | Descrição |
|---|---|---|
| `transactionId` | string | Identificador da transação reembolsada. |
| `status` | string | Status atualizado da transação. Um reembolso total bem-sucedido define este campo como `"REFUNDED"`. |
| `method` | string | Método de pagamento original (`"PIX"`, `"BOLETO"` ou `"CARD"`). |
| `amount` | integer | Valor original cobrado em centavos. |
| `refundedAmount` | integer | Valor total reembolsado em centavos. Em um reembolso total, é igual a `amount`. |
| `reason` | string | A justificativa fornecida na requisição, se houver. |
| `refundedAt` | string | Timestamp ISO 8601 de quando o reembolso foi processado. |

**Exemplo de Resposta**

```json
{
  "transactionId": "uuid-da-transacao",
  "status": "REFUNDED",
  "method": "CARD",
  "amount": 29700,
  "refundedAmount": 29700,
  "reason": "Solicitação do cliente",
  "refundedAt": "2026-06-26T14:32:00Z"
}
```

---

### Reembolso Parcial

Um reembolso parcial devolve ao comprador um valor específico — menor que a cobrança original. Inclua o campo `amount` com o valor exato em centavos que deseja reembolsar. É possível emitir múltiplos reembolsos parciais para a mesma transação, desde que o valor acumulado não ultrapasse a cobrança original.

**Campos da Requisição**

| Campo | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `transactionId` | string | ✅ | Identificador único da transação a ser parcialmente reembolsada. É o `transactionId` retornado por `POST /api/payment-gateway/process`. |
| `amount` | integer | ✅ | Valor a reembolsar em **centavos**. Deve ser maior que `0` e menor ou igual ao saldo reembolsável restante na transação. Exemplo: `10000` para R$ 100,00. |
| `reason` | string | ❌ | Explicação legível para o reembolso parcial. Armazenada para fins de auditoria e visível no painel da CertoPay. Exemplo: `"Desconto aplicado retroativamente"`. |

**Exemplo de Requisição**

```bash
curl -X POST https://v2.certopaybrasil.com/api/payment-gateway/refund \
  -H "X-Api-Key: sk_live_sua_chave_aqui" \
  -H "Content-Type: application/json" \
  -d '{
    "transactionId": "uuid-da-transacao",
    "amount": 10000,
    "reason": "Desconto aplicado retroativamente"
  }'
```

**Campos da Resposta**

| Campo | Tipo | Descrição |
|---|---|---|
| `transactionId` | string | Identificador da transação parcialmente reembolsada. |
| `status` | string | Status atualizado da transação. Um reembolso parcial bem-sucedido define este campo como `"PARTIAL_REFUND"`. |
| `method` | string | Método de pagamento original (`"PIX"`, `"BOLETO"` ou `"CARD"`). |
| `amount` | integer | Valor original cobrado em centavos. |
| `refundedAmount` | integer | Total acumulado reembolsado até o momento em centavos, incluindo este e quaisquer reembolsos parciais anteriores na mesma transação. |
| `reason` | string | A justificativa fornecida na requisição, se houver. |
| `refundedAt` | string | Timestamp ISO 8601 de quando este reembolso parcial foi processado. |

**Exemplo de Resposta**

```json
{
  "transactionId": "uuid-da-transacao",
  "status": "PARTIAL_REFUND",
  "method": "CARD",
  "amount": 29700,
  "refundedAmount": 10000,
  "reason": "Desconto aplicado retroativamente",
  "refundedAt": "2026-06-26T15:10:00Z"
}
```

> **Nota:** Reembolsos parciais definem o status da transação como `PARTIAL_REFUND`; reembolsos totais (ou uma série de reembolsos parciais que somam o valor original) definem como `REFUNDED`. Uma vez que a transação atinge o status `REFUNDED`, nenhum reembolso adicional pode ser emitido contra ela.

---

## Elegibilidade de Reembolso por Método de Pagamento

| Método | Reembolso Total | Reembolso Parcial | Observações |
|---|---|---|---|
| **CARD** | ✅ | ✅ | Reembolsos são processados imediatamente; o prazo de liquidação depende do banco emissor do comprador (normalmente 5–10 dias úteis). |
| **PIX** | ✅ | ✅ | Reembolsos via PIX (devoluções) são processados em tempo real pela infraestrutura do Banco Central do Brasil. |
| **BOLETO** | ✅ | ❌ | Reembolsos de boleto exigem que os dados bancários do comprador sejam fornecidos e são processados manualmente. Entre em contato com o suporte da CertoPay para instruções sobre reembolso de boleto. |

---

## Respostas de Erro

| Status HTTP | Significado |
|---|---|
| `400 Bad Request` | Um campo obrigatório está ausente ou o `amount` ultrapassa o saldo reembolsável. |
| `401 Unauthorized` | O cabeçalho `X-Api-Key` está ausente ou é inválido. |
| `404 Not Found` | Nenhuma transação foi encontrada para o `transactionId` fornecido. |
| `409 Conflict` | A transação já foi totalmente reembolsada ou está em um estado não reembolsável (ex.: `PENDING`, `DECLINED`). |
| `500 Internal Server Error` | Ocorreu um erro inesperado no lado da CertoPay. Tente novamente com recuo exponencial. |

> ⚠️ **Atenção:** Reembolsos só podem ser emitidos para transações em estado **capturado ou pago**. Tentar reembolsar uma transação `PENDING`, `DECLINED` ou já `REFUNDED` retornará um erro `409 Conflict`. Sempre verifique o status da transação via `GET /api/payment-gateway/transaction/{transactionId}` antes de iniciar um reembolso.
