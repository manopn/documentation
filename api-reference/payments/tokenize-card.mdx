# POST /api/payment-gateway/tokenize-card — Tokenizar um Cartão

**Descrição:** `POST /api/payment-gateway/tokenize-card` — Armazene um cartão como token reutilizável e cobre clientes recorrentes sem precisar manipular os dados brutos do cartão novamente.

---

O endpoint `/api/payment-gateway/tokenize-card` permite guardar com segurança o cartão de crédito de um cliente na CertoPay e receber em troca um `cardToken` opaco. Em toda cobrança subsequente desse cliente, passe o token no campo `cardToken` de `POST /api/payment-gateway/process` em vez de transmitir os dados brutos do cartão — mantendo as informações sensíveis do PAN fora dos seus servidores e simplificando seu escopo de conformidade com o PCI-DSS.

---

## URL Base

```
https://v2.certopaybrasil.com/api
```

## Endpoint

```
POST /payment-gateway/tokenize-card
```

---

## Cabeçalhos da Requisição

| Cabeçalho | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `X-Api-Key` | string | ✅ | Sua chave de API secreta. Deve ser mantida no servidor em todos os momentos. |
| `Content-Type` | string | ✅ | Deve ser `application/json`. |

---

## Corpo da Requisição

| Campo | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `customerId` | string | ✅ | Seu identificador interno para o cliente (UUID recomendado). A CertoPay associa o token resultante a este ID para que você possa gerenciar múltiplos cartões por cliente. |
| `doc` | string | ✅ | CPF do titular do cartão, somente dígitos. Exemplo: `"12345678900"`. |
| `holderName` | string | ✅ | Nome completo exatamente como aparece no cartão, em maiúsculas. Exemplo: `"JOAO DA SILVA"`. |
| `cardNumber` | string | ✅ | PAN do cartão com 16 dígitos, somente números. Exemplo: `"4111111111111111"`. |
| `brand` | string | ✅ | Bandeira do cartão. Valores aceitos: `"Visa"`, `"Mastercard"`, `"Elo"`. |
| `expirationMonth` | string | ✅ | Mês de vencimento com dois dígitos. Exemplo: `"12"`. |
| `expirationYear` | string | ✅ | Ano de vencimento com dois dígitos. Exemplo: `"28"` para 2028. |
| `securityCode` | string | ✅ | Código de segurança CVV/CVC do cartão. Exemplo: `"123"`. |
| `birthdate` | string | ✅ | Data de nascimento do titular no formato `YYYY-MM-DD`. Usada para verificação antifraude. Exemplo: `"1990-05-20"`. |
| `cellNumber` | string | ✅ | Número de celular do titular no formato brasileiro. Exemplo: `"(11) 99999-9999"`. |
| `address` | object | ✅ | Endereço de cobrança do titular. Deve corresponder ao endereço cadastrado no banco emissor para verificações AVS. |

**Campos do objeto `address`**

| Campo | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `postal_code` | string | ✅ | CEP brasileiro, somente dígitos. Exemplo: `"01310930"`. |
| `street` | string | ✅ | Nome da rua. Exemplo: `"Avenida Paulista"`. |
| `number` | string | ✅ | Número do imóvel. Exemplo: `"100"`. |
| `neighborhood` | string | ✅ | Bairro. Exemplo: `"Bela Vista"`. |
| `city` | string | ✅ | Nome da cidade. Exemplo: `"São Paulo"`. |
| `state` | string | ✅ | Código de duas letras do estado (UF). Exemplo: `"SP"`. |

---

## Exemplo de Requisição

```bash
curl -X POST https://v2.certopaybrasil.com/api/payment-gateway/tokenize-card \
  -H "X-Api-Key: sk_live_sua_chave_aqui" \
  -H "Content-Type: application/json" \
  -d '{
    "customerId": "uuid-do-customer",
    "doc": "12345678900",
    "holderName": "JOAO DA SILVA",
    "cardNumber": "4111111111111111",
    "brand": "Visa",
    "expirationMonth": "12",
    "expirationYear": "28",
    "securityCode": "123",
    "birthdate": "1990-05-20",
    "cellNumber": "(11) 99999-9999",
    "address": {
      "postal_code": "01310930",
      "street": "Avenida Paulista",
      "number": "100",
      "neighborhood": "Bela Vista",
      "city": "São Paulo",
      "state": "SP"
    }
  }'
```

---

## Campos da Resposta

| Campo | Tipo | Descrição |
|---|---|---|
| `cardToken` | string | Token opaco que representa o cartão armazenado. Este valor pode ser salvo com segurança no seu banco de dados e passado em futuras requisições `POST /api/payment-gateway/process` no campo `cardToken`. Ele não codifica nenhum dado recuperável do cartão. |

## Exemplo de Resposta

```json
{
  "cardToken": "tok_abc123..."
}
```

> 💡 **Dica:** Passe o valor de `cardToken` no lugar de todos os campos brutos do cartão (`cardNumber`, `cardHolderName`, `cardExpirationMonth`, `cardExpirationYear`, `cardSecurityCode`, `cardBrand`) em futuras chamadas a `POST /api/payment-gateway/process`. Isso elimina a necessidade de solicitar novamente os dados do cartão a clientes recorrentes e mantém sua integração fora do escopo de armazenamento de dados do PCI-DSS.

---

## Respostas de Erro

| Status HTTP | Significado |
|---|---|
| `400 Bad Request` | Um campo obrigatório está ausente ou malformado. Verifique o array `errors` no corpo da resposta para detalhes por campo. |
| `401 Unauthorized` | O cabeçalho `X-Api-Key` está ausente ou é inválido. |
| `422 Unprocessable Entity` | Os dados do cartão foram rejeitados durante a validação (ex.: cartão vencido, CVV inválido). |
| `500 Internal Server Error` | Ocorreu um erro inesperado no lado da CertoPay. Tente novamente com recuo exponencial. |

> ⚠️ **Atenção:** O endpoint de tokenização deve ser chamado apenas a partir do seu **servidor**. Nunca o chame diretamente de um navegador ou aplicativo mobile, pois isso exporia sua `X-Api-Key`. Use um formulário no lado do cliente para coletar os dados do cartão e envie-os via POST ao seu próprio backend, que então realiza a chamada à CertoPay.
